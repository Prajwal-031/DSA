# Tree Practice Problems - Interview Preparation

## Problem list (by difficulty)

### Easy (start here)

1. **Maximum Depth of Binary Tree** — LeetCode #104
   - Pattern: recursive DFS for height
   - Time: O(n), Space: O(h)

2. **Invert Binary Tree** — LeetCode #226
   - Pattern: recursive swap of left/right subtrees
   - Time: O(n), Space: O(h)

### Medium (build skills)

1. **Binary Tree Inorder Traversal** — LeetCode #94
   - Pattern: iterative stack-based traversal
   - Time: O(n), Space: O(h)

2. **Symmetric Tree** — LeetCode #101
   - Pattern: recursive check for mirror symmetry
   - Time: O(n), Space: O(h)

### Hard (master level)

1. **Binary Tree Maximum Path Sum** — LeetCode #124
   - Pattern: recursive path sum with global max
   - Time: O(n), Space: O(h)

---

## Patterns to master for Tree

1. **DFS Traversals** — Pre/in/post-order for different access patterns (94, 144, 145).
2. **BFS/Level Order** — Use queue for level-by-level processing (102, 107).
3. **Recursion with State** — Track path, sum, or symmetry in subtrees (101, 124).

---

## Interview tips

- State the **definition** of Tree clearly in one sentence, then give one concrete example.
- Always mention **edge cases** before coding (empty tree, single node, skewed tree).
- For runtime, say **worst case** unless the interviewer asks for average.
- Practice **explaining** your solution in two minutes without code.
