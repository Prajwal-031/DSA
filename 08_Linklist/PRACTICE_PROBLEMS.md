# Linklist Practice Problems - Interview Preparation

## Problem list (by difficulty)

### Easy (start here)

1. **Reverse Linked List** — LeetCode #206
   - Pattern: iterative reversal with three pointers
   - Time: O(n), Space: O(1)

2. **Merge Two Sorted Lists** — LeetCode #21
   - Pattern: two pointers merge
   - Time: O(n + m), Space: O(1)

### Medium (build skills)

1. **Linked List Cycle** — LeetCode #141
   - Pattern: fast/slow pointers (Floyd's cycle detection)
   - Time: O(n), Space: O(1)

2. **Linked List Cycle II** — LeetCode #142
   - Pattern: fast/slow + cycle start detection
   - Time: O(n), Space: O(1)

### Hard (master level)

1. **Merge k Sorted Lists** — LeetCode #23
   - Pattern: priority queue or divide-and-conquer merge
   - Time: O(n log k), Space: O(k) or O(1)

---

## Patterns to master for Linklist

1. **Two Pointers** — For merging, finding middle, or detecting cycles (e.g., 21, 141).
2. **Fast/Slow Pointers** — Cycle detection and finding middle (141, 876).
3. **Reversal** — In-place reversal for problems like 206 or palindrome checks.

---

## Interview tips

- State the **definition** of LinkedList clearly in one sentence, then give one concrete example.
- Always mention **edge cases** before coding (empty list, single node, cycles).
- For runtime, say **worst case** unless the interviewer asks for average.
- Practice **explaining** your solution in two minutes without code.
